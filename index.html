learning git

  1. by default, git ki main branch master branch hoti hai. master branch wala code hi aage publish hota hai production me
  2. so we dont work directly with master branch. only few have access to master branch
  3. so we create new branch which have code of master branch plus additonal code. we work there and later merge that with master branch
  4. to see current branches command is git branch
  5. to make new branch, command is git branch branch-name. eg. git branch about
  6. git branch jab krenge toh jis branch me hum hain, uska color change ho jaayegaa
  7. to come into master branch again, type git checkout master
  8. to go into about branch, command is git checkout about
  9. jab hume lagta hai ki sab kuch shi chal raha hai choti wali branch me , toh hum use merge kr dete hain master branch me , we will see later
  10. whatever changes we do in choti wali branch will not be affected in master branch. master branch kaa access sirf basde logo ke pass hota hai. wahi log commit kr sakte hain
  11. agar hame merge krwana hai toh hum pull request send krte hain, we will see later
  12. abhi toh we have all access in local, toh kahi bhi merge aur kuch bhi kro
  13. git log
  14. git log --all. all commit done in our repo
  15. to see this in short do git log --all --oneline
  16. HEAD basically repreent ki hum kis branch me hai abhi, that is hamari working directory
  17. hamne joh new branch banayi hai , uske andar master branch ka sab kcuh hogaa, commit , code etc. bus jab aage se new branch me commit krenge toh woh sir us new branch me honge, master branch me reflect nhi honge
  18. git branch --delete branch-name. this is how to delete a branch. this will work if we have not committed anything in the branch
  19. but agar hume ese branch ko delete krna hai jiske andar commit hai toh command is git branch --D branch-name.
  20. kisi bhi branch ko delete krne ke liye usse bahar jaana hoga
